## Project Overview

React Native mobile wallet app for iOS/Android using **Expo SDK 54**. Shares core wallet SDK with the Chrome extension (`../src/`). The mobile app is in `mobile-wallet/`.

### Tech Stack
- **Framework:** Expo SDK 54 + React Native
- **Routing:** expo-router (file-based)
- **Styling:** NativeWind (Tailwind for RN)
- **State:** Zustand (`store/walletStore.ts`)
- **Storage:** expo-secure-store (iOS Keychain), AsyncStorage
- **Crypto:** Pure JS implementations using @noble/secp256k1, @noble/hashes

## Key Implementation Details

### 1. Pure JS Crypto (Critical)

React Native doesn't support WebAssembly or Node crypto. All crypto is implemented in pure JS:

| File | Purpose |
|------|---------|
| `stubs/tiny-secp256k1.js` | secp256k1 using @noble/secp256k1 |
| `stubs/bip32.js` | BIP32 HD key derivation |
| `stubs/ecpair.js` | Bitcoin ECPair implementation |
| `stubs/bitcoin-index.js` | BIP-84 Native SegWit address derivation + BitcoinProvider stub |

**Metro config** (`metro.config.js`) redirects imports:
```js
'@wallet/bitcoin/index.js': './stubs/bitcoin-index.js',
'bip32': './stubs/bip32.js',
'tiny-secp256k1': './stubs/tiny-secp256k1.js',
```

### 2. SDK Integration

The app imports from `../src/` via Metro path alias `@wallet/*`:
```js
import { WalletAppService } from '@wallet/app-service.js';
```

**WalletBridge** (`services/WalletBridge.ts`) wraps `WalletAppService` and handles:
- Wallet creation/import with password encryption
- Network switching (returns network-specific address)
- Balance refresh with price fetching
- Master password session management
- **NEW:** Transaction history fetching for all chain types

### 3. Master Password System

Users set one password that unlocks all wallets. Stored as session hash in memory:
- `walletStore.masterPasswordHash` - SHA256 of password (session only)
- `unlock()` validates password against first wallet, caches hash
- `create()/import()` reuses session password for new wallets

### 4. Network-Specific Addresses

When switching networks, the correct address type is derived:
- **EVM:** Standard Ethereum address (same across all EVM chains)
- **Bitcoin:** BIP-84 Native SegWit (bc1...)
- **Solana:** Ed25519 base58
- **XRP:** secp256k1 with X-address support

`switchNetwork()` returns `{ address }` which updates the store.

### 5. Price Service

`services/price-service.ts` fetches from CoinGecko:
- Rate limited with local cache (60s TTL)
- Falls back to cached prices on error
- Supports batch fetching for multiple tokens

### 6. Transaction History (NEW)

**WalletBridge.getTransactions()** routes to the appropriate explorer API based on network type:

| Network Type | Explorer API | Source |
|--------------|--------------|--------|
| EVM | `ExplorerAPI` | Etherscan V2 |
| Bitcoin | `BitcoinProvider` | Mempool.space |
| Solana | `SolanaExplorer` | Solana RPC |
| XRP | `XRPProvider` | XRP Ledger RPC |

**walletStore** manages transaction state:
- `transactions[]` - Fetched transaction list
- `transactionFilter` - 'all' | 'sent' | 'received'
- `loadTransactions()` - Fetches from WalletBridge
- `getFilteredTransactions()` - Returns filtered list

**Activity Screen** (`app/(tabs)/activity.tsx`):
- Pull-to-refresh with RefreshControl
- Filter chips (All/Sent/Received)
- Tap transaction to open details modal
- Status badges (Pending/Failed)
- Relative timestamps

**TransactionDetailsModal** (`components/TransactionDetailsModal.tsx`):
- Full transaction details display
- Copy-to-clipboard for addresses/hash
- "View in Explorer" button opens block explorer

## Known Issues

### 1. Bitcoin Balance Shows "Error"
The `BitcoinProvider` stub in `stubs/bitcoin-index.js` has placeholder explorer that returns 0. Need to implement real API calls or connect to actual `BitcoinExplorer`.

**Fix:** Update `BitcoinExplorer` class in stub to call real Blockstream/Mempool API.

### 2. Token/Network Icons Missing
Currently using letter-based fallback (first letter of symbol in colored circle). The `assets/crypto/` folder has SVGs but React Native needs a transformer.

**Options:**
1. Use `react-native-svg` with metro SVG transformer
2. Convert SVGs to PNGs
3. Keep using the letter-based `CryptoIcon` component with brand colors

### 3. Account Index Not Persisted
When creating multiple accounts, the `currentAccountIndex` resets on app restart. Need to persist to storage.

### 4. EVM Transaction History Needs Network Registration
The ExplorerAPI may need networks registered before fetching transactions. The WalletBridge handles this automatically but some networks might not have `explorerApiUrl` configured.

## Running the App

```bash
cd mobile-wallet
npm install
npx expo start --clear
```

Press `i` for iOS simulator or `a` for Android emulator.

### Build for Device
```bash
npx expo run:ios     # iOS device/simulator
npx expo run:android # Android device/emulator
```
