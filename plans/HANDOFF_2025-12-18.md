## 1. Price Provider Refactor

### Problem
CoinGecko free tier was causing frequent failures due to strict rate limits (~10-30 calls/minute).

### Solution
Created a pluggable provider architecture with:
- **CoinPaprika** as primary (20K calls/month free tier)
- **CoinGecko** as fallback (wider token support, contract lookups)

### New Files Created

**Core SDK (`src/price-providers/`):**

| File | Purpose |
|------|---------|
| `types.ts` | `PriceProvider` interface, shared types (`TimeRange`, `PricePoint`, etc.) |
| `coinpaprika.ts` | Primary provider implementation (priority 1) |
| `coingecko.ts` | Fallback provider implementation (priority 2) |
| `provider-manager.ts` | Fallback logic, caching (1min/5min/1hr TTLs) |
| `index.ts` | Singleton manager, provider registration |

**Tests:**

| File | Description |
|------|-------------|
| `tests/price-providers.test.js` | 15 unit tests (all passing) |

### Modified Files

| File | Changes |
|------|---------|
| `src/price-service.ts` | Now delegates to provider manager; added `getPriceHistory()` and `getTokenMetadata()` exports |
| `mobile-wallet/services/price-history.ts` | Rewritten to use CoinPaprika primary + CoinGecko fallback |

### Architecture

```
┌─────────────────┬─────────────────┬─────────────────┐
│       CLI       │    Extension    │     Mobile      │
└────────┬────────┴────────┬────────┴────────┬────────┘
         │                 │                 │
         ▼                 ▼                 ▼
┌─────────────────────────────────────────────────────┐
│              src/price-service.ts                   │
│  (Backwards-compatible API + new history methods)   │
└───────────────────────┬─────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────┐
│         src/price-providers/provider-manager.ts     │
│              (Fallback logic + caching)             │
└───────────────────────┬─────────────────────────────┘
          ┌─────────────┴─────────────┐
          ▼                           ▼
   ┌──────────────┐          ┌──────────────┐
   │ coinpaprika  │          │  coingecko   │
   │  (Primary)   │          │  (Fallback)  │
   └──────────────┘          └──────────────┘
```

### How to Change Provider Priority

In `src/price-providers/index.ts`:

```typescript
// Current (CoinPaprika primary):
priceProviderManager.registerProvider(new CoinPaprikaProvider()); // Priority 1
priceProviderManager.registerProvider(new CoinGeckoProvider());   // Priority 2

// To make CoinGecko primary, swap the order or change priority values
```

## 2. Chart Y-Axis Labels

### Problem
The price chart had no Y-axis labels, making it hard to understand price scale.

### Solution
Added Y-axis with:
- 4 evenly-spaced price ticks
- Horizontal dashed grid lines
- Compact price formatting ($3.5K, $1.2M, etc.)

### Modified File

`mobile-wallet/components/PriceChart.tsx`

### Changes Made

1. **Increased left padding** from 8px to 55px for label space
2. **Added `SvgText`** import from react-native-svg
3. **Added `getYAxisTicks()`** - calculates 4 evenly-spaced values
4. **Added `formatAxisPrice()`** - compact price formatting
5. **Added Y-axis rendering** - grid lines + labels in SVG
